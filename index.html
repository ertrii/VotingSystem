Voting System vs. 1.0.0

Designed to improve the user experience (player) of any MapleStory server and eliminate the abusive votes and typical votes in exchange for nx (cash)

TEMPLATE TESTING

Introdución.

Gracias por su interéz. Estoy seguro que le será de mucha ayuda.

Muchas veces se ha visto la existencia de dos o mas usuarios en un solo hogar y el problema que se generalizaba para realizar dichos votos debido al registro de ip por la necesidad de nx. Quise eliminar este tipo de sistema.

El objetivo de esto es que los usuarios obtengan nx por otros medios y que se vuelva un reto. Esto se desarrollo con la idea de que obtengan nx(card etc) por ciertos mobs para aumentar la actividad en mapas que ocupan ellos como sleepywood (hablo en caso de versiones antiguas) y evitar los votos abusivos.

Este sistema no reducirá la actividad de votos ya que esto se diseño pensando en estos detalles tambien. No se preocupe, se lo aseguro.

Los usuarios pueden recibir a cambio por los votos otros premios escogidos por los miembros del staff. No sería genial que un usuario antiguo gane por ejemplo una medalla(Medal Equip) como reconocimiento por haber logrado acumular tantos votos, sería genial. Aparte de esto, visualizar su posición en el ranking.

Requerimiento.

  • PHP versión 5.5+
  • Mysql versión 5.0+
  • Base de Datos del servidor MapleStory para obtener datos necesarios para el sistema.

Detalles

  • Para realizar un voto sin afectar al otro en un hogar usted tendrá que editar el campo ip_control a 0 en la tabla votes del usuario a quien desee. Esto causará que no registre la ip.
  • La function setGameItems de connect.php, line 196, no está del todo probado debido a la falta de testing en el juego. Ayudame a mejorarlo
  • Para las sesiones, el sistema necesitará tomar la id del usuario de la base de datos del juego
  • Los Items se agregaran al primer character de su cuenta. En caso de que no exista le pedirá que se cree uno. Si desea que los items ganados vaya al otro character de su cuenta tendrá que iniciar sessión para activar las configuraciones de vote.
  • El sistema está empleado para registrar los datos necesarios del usuario en su base de datos cuando hay echo su primer voto.
  • No es sensible a mayúsculas y minúsculas pero si deseas que toda entrada sea minúscula puedes descomentar line 165 vote.php, esto solo afectara al primer formulario(formVote).

Configuration

En el archivo config.php existen muchas opciones (const):

  • Voting System (boolean), Activar o desactivar el sistema
  • System Off (string), si el sistema esta desactivado mostrará un mensaje de mantenimiento.
  • Host (string).
  • DB_user (string), nombre de usuario.
  • DB_pass (string), password.
  • DB_vote (string), nombre de la base de datos del voting system.
  • DB_maple (string), nombre de la base de datos de su servidor.
  • SESSION_REQUERID (boolean), activalo si deseas que inicien sesión primero
  • Session Variable (string), nombre de la variable de $_SESSION
  • Timezone (string), http://php.net/manual/es/timezones.php
  • Time for the next vote (int), hora, tiempo para el siguiente voto.
  • Minimo Level Requerid (int), basta con un char
  • Ip Control (boolean), registro de ip. para un voto en solo un hogar.
  • Additional Vote (boolean), en caso de que desees que el usuario acumule votos aparte de los votos totales. Esto es para Items type accumulate(a).
  • TOPSITE_URL (string), link de la página a donde se redirigirá despues del voto.
  • RANK_TABLE_ROWS(int), columnas del ranking.
  • RANK_TABLE_COL_START_DATE (boolean), mostrar campo start_date
  • RANK_TABLE_COL_VOTE_ADDITIONAL (boolean), mostrar campo vote_additional
  • Message, Atención con Vote_Notice, existe una etiqueta span con una id, “v-count_time”, este es tomado por un script para realizar dicho conteo para luego redireccionar, si usted no lo desea puede eliminar, line 101 vote.php.

Item

Los Items se les conoce como premios(PRIZE), usted puede editarlos o eliminarlos, desactivar y tambien agregar nuevos premios, recuerde seguir la estructura debida.

  • DIRECTORY, ruta donde se encuentra la imagen.
  • Status (boolean), activar o desactivar el premio
  • Type (string), tipos de premio, “continuous”, “requerid” y “accumulate”, todo en minuscula.
    • Continuous(c), El usuario obtendra el premio de manera continua.
    • Requerid(r), El usuario obtendra el premio solo cuando aya llegado a ese voto requerido.
    • Accumulate(a), El usuario obtendra el premio al acumular el voto exacto aparte del voto total ya acumulado.
  • Quantity (int, array(int), array(array(min(int) , max(int)))), cantidad de items por voto, en caso de ser un valor numerico, será una cantidad estatica, en caso de ser un array, ejemplo [2, 5] estará sincronizado con el voteRequerid que veremos mas adelante. Si usted desea darle mas dinamismo como algo random usted debera seguir esta estructura. 
    
                            'quantity'      =>      [
                                array('min' => 0, 'max' => 2),
                                array('min' => 2, 'max' => 5),
                                array('min' => 3, 'max' => 6),
                                array('min' => 1, 'max' => 7)
                            ]
    o tambien 
    
                            'quantity'      =>      [
                                3,
                                array('min' => 1, 'max' => 1),
                                5
                            ]
    min y max, son valores que se le asiganará a una function rand, line 23, 30 reward.php.
  • Item(array), datos necesarios del item.
    • Id (int), identificador del item del juego.
    • Name(string), nombre el item.
    • Img(string), nombre de la imagen del item, no olvide la extensión.
    Usted puede agregar mas propiedades.
  • voteRequerid (array), valores numerico que se requiere para los votos, sus comportamientos será diferente dependiendo por el Type escrito.

Usted puede agregar mas items siempre y cuando cumpla con los standares que se le ha informado. Una vez echo esto debe agregar en el retorno de la function get.

El sistema automáticamente estará extrayendo a su base de datos los id account, id characters y ip cuando el usuario haya logrado hacer el voto. No se preocupe usted. Luego del voto se insertará el item en la tabla del juego, inventoryitems.

Security

Tambien pense en esto. Cualquier intento forzado de inyección como, mysql query, caracteres especiales hasta una simple comilla será retenida. Todo intento será registrado en la base de datos en la tabla filter; ip, fecha y el dato escrito, en caso de existir una sesión tambien se le registrará. Al intento de ingreso de un character por defecto no existente mostrará un mensaje de error e1000

Al votar con un character por defecto no existente el sistema no dará error simplemente mostrará un aviso.
